The UGO & RWX Foundation

Linux permissions are grouped into three categories: User (u), Group (g), and Others (o). For each, you define Read (r=4), Write (w=2), and Execute (x=1).

Modifying Permissions (chmod)

# 1. Symbolic Mode (Add/Remove)
$ chmod u+x script.sh
// Adds execute (x) to User (u)
$ chmod g-w,o=r report.txt
// Removes write from Group, sets Others to strictly read-only
# 2. Octal/Numeric Mode (Absolute)
$ chmod 755 public_html/
// User: 7 (4+2+1 = rwx)
// Group: 5 (4+1 = r-x)
// Others: 5 (4+1 = r-x)

Ownership (chown & chgrp)

# Change File Owner
$ sudo chown alice project.txt
// Makes user 'alice' the owner
# Change Owner AND Group simultaneously
$ sudo chown alice:devteam source_code/
// Sets owner to 'alice', group to 'devteam'
# Change Group explicitly
$ sudo chgrp finance budget.xlsx
// Sets the group owner to 'finance'
# Recursive changes (Apply to directories)
$ sudo chown -R bob:managers /var/data/
// Modifies /var/data and all files inside it

System Overview: The Exception, Not the Rule

In Red Hat Enterprise Linux (RHEL), certain scenarios require elevated or restricted execution contexts beyond standard UGO. Special permissions exist for these edge cases. Out of thousands of files on a standard RHEL system, only a tiny fraction utilize these bits.

SUID (Set User ID)

Mostly on essential binaries (e.g., passwd). Allows regular users to execute them with the owner's (usually root) privileges.

SGID (Set Group ID)

Used on collaborative directories. New files inherit the directory's group ownership rather than the creator's group.

Sticky Bit

Applied to world-writable directories (e.g., /tmp). Prevents users from deleting each other's files.

Set User ID (SUID)

When SUID is set, the file runs with the privileges of the file's owner. Meaningless on directories.

  • Apply via Math: chmod 4755 file
  • Apply via Symbol: chmod u+s file
  • Symbolic Output: Replaces User's x with s.
  • Edge Case: Without owner execution, it shows a capital S.
$ ls -l /usr/bin/passwd
-rwsr-xr-x. 1 root root 34K Nov /usr/bin/passwd

Set Group ID (SGID)

On files, runs with group privileges. On directories, new files inherit the directory's group.

  • Apply via Math: chmod 2775 dir/
  • Apply via Symbol: chmod g+s dir/
  • Symbolic Output: Replaces Group's x with s.
  • Edge Case: Without group execution, it shows a capital S.
$ ls -ld /var/www/html/devteam
drwxr-wsr-x. 2 admin dev_group 4.0K /var/www/html/devteam

The Sticky Bit

Used on directories. Only the file owner, directory owner, or root can delete/rename files inside.

  • Apply via Math: chmod 1777 dir/
  • Apply via Symbol: chmod +t dir/
  • Symbolic Output: Replaces Others' x with t.
  • Edge Case: Without others execution, it shows a capital T.
$ ls -ld /tmp
drwxrwxrwxt. 15 root root 4.0K Oct 12 /tmp

Access Control Lists (ACLs)

Standard UGO permissions have a limitation: what if you want to give a specific user (who isn't the owner) read access, without giving it to the entire group or everyone else? ACLs solve this by allowing fine-grained permissions for specific users and groups.

Crucial Indicator: When an ACL is active on a file, the standard ls -l output will show a + sign at the end of the permissions block (e.g., -rw-rwxr--+).

Viewing ACLs (getfacl)

$ getfacl document.txt
# file: document.txt
# owner: root
# group: root
user::rw-
user:alice:rwx // Specific user ACL
group::r--
group:devteam:rw- // Specific group ACL
mask::rwx
other::r--

Modifying ACLs (setfacl)

# Grant user 'alice' read/write access
$ setfacl -m u:alice:rw file.txt
# Grant group 'contractors' read access
$ setfacl -m g:contractors:r data/
# Remove a specific ACL rule for 'alice'
$ setfacl -x u:alice file.txt
# Remove ALL ACLs (revert to basic UGO)
$ setfacl -b file.txt
# Set Default ACL for new files in a directory
$ setfacl -m d:u:alice:rwx project_dir/
